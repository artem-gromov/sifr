# Sifr — план улучшений

## Проект
Кросс-платформенный менеджер паролей (v1.0.1) с зашифрованным SQLCipher vault. Архитектура: `sifr-core` (бизнес-логика) + `sifr` CLI (Ratatui TUI) + нативные клиенты через UniFFI.

---

## Высокий приоритет

### 1. Schema Migrations Infrastructure
**Проблема:** `db::migrate()` — пустая заглушка. Нет способа эволюционировать схему после v1.
**Решение:** Реализовать migration runner в `db/mod.rs` через таблицу `schema_version`.

### 2. Auto-Lock After Inactivity
**Проблема:** Vault остаётся разблокированным до явной блокировки (`L`).
**Решение:** Добавить настраиваемый таймаут неактивности (5/10/15 мин) с автоблокировкой.

### 3. Password Strength Indicator
**Проблема:** Генератор паролей работает, но при ручном вводе пароля нет обратной связи.
**Решение:** Индикатор силы (weak/medium/strong) на основе entropy calculation в форме entry.

---

## Средний приоритет

### 4. Favorites First + Sorting
**Проблема:** `list_entries()` сортирует только по `title ASC`. Избранные записи не выделены.
**Решение:** Избранные всегда показываются первыми (в рамках текущего search filter). Сохранять последний filter/sort в config.

### 5. CLI Test Coverage
**Проблема:** `core/tests/vault_test.rs` покрывает core. CLI (`cli/src/`) не покрыт тестами.
**Решение:** Добавить интеграционные тесты для ключевых CLI flows (vault create/open, entry CRUD).

### 6. Vault Import/Export
**Решение:**
- CSV/JSON import (bulk создание entries)
- JSON export для backup

### 7. Concurrent Access Handling (SQLite + SQLCipher)
**Проблема:** Необходимо проверить как работает concurrent access в SQLite/SQLCipher (WAL mode).
**Решение:** Исследовать поведение при одновременном открытии vault из нескольких процессов. Добавить advisory locking или документировать ограничения.
